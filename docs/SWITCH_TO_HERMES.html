Why Hermes?

  • Self-improving agent: creates and updates skills from experience
  • Better memory compression and context management
  • minimax-portal/MiniMax-M2.7 via Hermes (same model, different integration)
  • Skills auto-improve during use
  • hermes update to stay current

Quick Start

Check Hermes Status

hermes status
hermes gateway status

Start Hermes Gateway

# From ~/.hermes/ directory:
hermes gateway run        # foreground (for testing)
# OR install as service:
hermes gateway install     # user-level service
sudo hermes gateway install --system  # system-level

Stop Hermes Gateway

hermes gateway stop
# Or kill manually:
ps aux | rg 'hermes gateway' | rg -v rg | awk '{print $2}' | xargs kill

Verify Hermes is Responding

DM the OpenClaw prod bot on Slack (Hermes now controls it).

Preserving the OpenClaw Path

The OpenClaw gateway (ai.openclaw.gateway) continues running for:

  1. AO automation: skeptic-cron merge gates, PR reactions (posts to staging Slack app)
  2. OpenClaw cron jobs: ~/.openclaw/cron/jobs.json
  3. Agent orchestrator: spawns, lifecycle workers
  4. mem0 memory: OpenClaw's Qdrant-backed memory store

Verify OpenClaw is Running (AO path)

curl -fsS -m 8 http://127.0.0.1:18789/health
# Expected: HTTP 200 (JSON body may vary by runtime/version)

openclaw gateway health --timeout 30000
# Slack: ok — staging Slack app connected

If OpenClaw needs restarting:

launchctl kickstart -kp gui/$(id -u)/ai.openclaw.gateway

Hermes Configuration

.env Slack tokens (prod bot)

# In ~/.hermes/.env:
SLACK_BOT_TOKEN=<SLACK_BOT_TOKEN>
SLACK_APP_TOKEN=<SLACK_APP_TOKEN>
GATEWAY_ALLOW_ALL_USERS=true   # Required — allows DM access

OpenClaw uses staging Slack tokens

# In ~/.openclaw/openclaw.json — channels.slack:
{
  "botToken": "<SLACK_BOT_TOKEN>",
  "appToken": "<SLACK_APP_TOKEN>"
}

Switching Model

Hermes uses minimax-portal/MiniMax-M2.7 by default. To change:

hermes model

To switch back to OpenClaw as primary agent: stop Hermes gateway, DM the OpenClaw bot.

Troubleshooting

Hermes gateway won't start

hermes gateway status
hermes doctor
cat /tmp/hermes-gateway.log   # or ~/.hermes/logs/gateway.log

Slack not responding

  • Verify Hermes is running: ps aux | rg 'hermes gateway' | rg -v rg
  • Check Slack tokens are set: rg 'SLACK_BOT_TOKEN' ~/.hermes/.env
  • GATEWAY_ALLOW_ALL_USERS=true must be in ~/.hermes/.env

OpenClaw AO automation broken

curl -fsS -m 8 http://127.0.0.1:18789/health
openclaw gateway health --timeout 30000
