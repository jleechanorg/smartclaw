# PR #130 Design: Companion World Events — Phase 2

## Summary
Phase 2 of the Living Companions roadmap. Adds `companion_joined`, `companion_departed`, and `companion_acted` event types to `WorldEvent`. Wires `generateCompanionEvents()` into `runSimulation()` to produce player-visible world events from companion actions.

## Problem
Companion actions (join, depart, act) were not surfaced as `WorldEvent` entries in the simulation output. Players could not see companion activity in the world events feed.

## Goals
- Add `companion_joined`, `companion_departed`, `companion_acted` to `CompanionEventType`
- `generateCompanionEvents()` maps companion action results to world events
- Wire into `faction_simulator.ts` after each companion tick
- Tests in `companion_events.test.ts` and `companion_world_events.test.ts`

## Changes
- **`packages/backend/src/world/events.ts`**: Added `companion_joined`, `companion_departed`, `companion_acted` to `CompanionEventType` union. Extended `companionActionEventMap` with mappings for these new types.
- **`packages/backend/src/world/faction_simulator.ts`**: Integrated companion event generation in the companion tick processing loop (11 additions, 9 deletions). Events appended to `state.pendingWorldEvents` using `addWorldEvent()`.
- **`packages/backend/src/world/faction_types.ts`**: Added companion event type fields to faction state types (4 additions, 1 deletion).
- **`packages/backend/src/world/types.ts`**: Extended `WorldEvent.type` union with `companion_joined`, `companion_departed`, `companion_acted` (2 additions, 1 deletion).
- **`packages/backend/src/world/companion_events.test.ts`**: Unit tests for companion event generation (100 lines added).
- **`packages/backend/src/__tests__/companion_world_events.test.ts`**: Integration tests for world event generation from companion ticks (159 lines added).

## Key Design Decisions
- New companion event types extend the existing `WorldEvent` union
- The 50-event cap on `pendingWorldEvents` is enforced by `addWorldEvent()` in `faction_store.ts` (no duplicate cap logic in `faction_simulator.ts`)
- Severity mapping: `companion_joined`=low, `companion_departed`=low, `companion_acted`=medium
- Tests cover all mapped action types (combat, explore, trade, quest, guard, social, travel)
