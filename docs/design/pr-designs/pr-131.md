# PR #131 Design: Basic Companion State Display in UI — Phase 1

## Summary
Phase 1 of the Living Companions roadmap. Surfaces companion state (name, mode, location, last action) in the existing game UI using the `CompanionPanel` component.

## Problem
Companions exist in the backend but their state is not visible to the player in the web UI.

## Goals
- Display companions list in the existing game view
- Show: name, mode (archetype/role), location, and last action summary
- Surface `lastAction` from the backend (most recent action)
- Use existing `CompanionPanel` component as the rendering layer

## Changes
- **`packages/backend/src/__tests__/companions_ui_api.test.ts`**: Integration tests verifying the companions list API endpoint returns `lastAction` (91 lines added).
- **`packages/backend/src/app.ts`**: Updated companions list endpoint to include `lastAction` field in the API response (6 additions, 1 deletion).
- **`packages/backend/src/storage/sqlite_repository.ts`**: Added `getLatestActionsForCompanions(companionIds: string[])` method. Uses a single SQL query with `ROW_NUMBER() OVER (PARTITION BY companion_id ORDER BY timestamp DESC, rowid DESC)` to batch-fetch the most recent action per companion (37 lines added).
- **`packages/web/src/App.tsx`**: Wired companions data fetch and state management to pass `lastAction` to components (18 additions, 1 deletion).
- **`packages/web/src/components/CompanionPanel.tsx`**: Updated to display `lastAction.summary` alongside name, mood, location (5 additions, 1 deletion).
- **`packages/web/src/screens/GameScreen.tsx`**: Integrated `CompanionPanel` into the game screen layout (9 additions, 1 deletion).
- **`packages/web/src/types/game.ts`**: Added `lastAction` field to `Companion` type definition (1 addition).

## Key Design Decisions
- `getLatestActionsForCompanions()` is a batch method (single query) to avoid N+1 queries
- Uses SQL `ROW_NUMBER()` window function to get the most recent action per companion
- `lastAction` reflects the absolute latest action regardless of type (including `history_summary` records)
- Falls back to `null` if no actions exist yet for a companion
- Both backend and frontend changes included in this single PR
- Integration tests use a real SQLite backend (no mocks)
