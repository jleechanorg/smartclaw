# PR #129 Design: Companion Party Management (Add/Remove) — Phase 1

## Summary
Adds comprehensive integration tests for the companion party management endpoints (Phase 1 of worldai_claw roadmap). The endpoints `POST /sessions/:id/companions/:companionId/join` and `DELETE /sessions/:id/companions/:companionId/leave` are tested here via TDD.

## Problem
Companion party management is part of the worldai_claw roadmap Phase 1. TDD requires tests to be written first, then the implementation follows.

## Goals
- TDD: Write tests for companion party management endpoints
- Validate all happy paths, error handling, edge cases, and IDOR prevention
- Full coverage of join/leave operations with security checks

## Changes
- **`packages/backend/src/__tests__/companion_party_management.test.ts`**: New integration test file. Tests: join success, duplicate join 409, leave success, leave nonexistent 404, IDOR 403 for wrong session.

## Key Design Decisions
- This PR is tests-only (TDD step): the implementation of `join` and `leave` endpoints is in the backing storage layer
- Test covers: happy-path join, idempotency (duplicate join → 409), leave success, leave nonexistent → 404, cross-session IDOR → 403
- Tests exercise the actual running server against a real SQLite database
