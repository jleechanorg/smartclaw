# PR #128 Design: Companion Edit Notifications via SSE

## Summary
Implements real-time SSE (Server-Sent Events) notifications for DM companion force-edits. Adds a `/sessions/:id/events` endpoint that streams `companion_edited_by_dm` events.

## Problem
DM companion force-edits happened silently with no real-time notification to connected clients.

## Goals
- Session-level SSE endpoint for real-time event streaming
- Emit `companion_edited_by_dm` events when a DM force-edits a companion
- Clean resource release on client disconnect

## Changes
- **`packages/backend/src/services/sessionEventBus.ts`**: New module providing a per-session event emitter for SSE streams. Manages client registration and cleanup.
- **`packages/backend/src/app.ts`**: Added `GET /sessions/:id/events` and `GET /api/sessions/:id/events` SSE endpoints. Emits `companion_edited_by_dm` event inside the force-edit handler (`POST /sessions/:id/dm/companions/:companionId/force-edit`) after a successful update.
- **`packages/backend/src/__tests__/companion_notifications.test.ts`**: Integration tests verifying SSE endpoint returns 200, event is emitted on force-edit, connection cleanup on close.

## Key Design Decisions
- Per-session EventEmitter avoids cross-session leaks
- SSE uses `text/event-stream` with heartbeat comments to keep connections alive
- `companion_edited_by_dm` event payload: `{companionId, field, oldValue, newValue, dmId, reason}`
- Dual route registration: `/sessions/:id/events` and `/api/sessions/:id/events` for API prefix compatibility
- Cleanup on `response.on('close')` removes the listener to prevent memory leaks
- Force-edit endpoint: `POST /sessions/:id/dm/companions/:companionId/force-edit`
