Incident: OpenClaw 2026.3.28 ws-stream Protocol Regression

Date: 2026-03-29 Severity: P0 Status: Resolved Bead: orch-420

Summary

OpenClaw 2026.3.28 introduced a protocol version mismatch in the embedded agent WebSocket stream (ws-stream) that caused all WebSocket upgrade attempts to fail with HTTP 500. The constant retry storm (367 failed connections/day) starved the Node.js event loop, causing Slack Socket Mode pong timeouts and dropped Slack replies.

Root Cause

Primary: @agentclientprotocol/sdk version mismatch

OpenClaw 2026.3.28 upgraded @agentclientprotocol/sdk from 0.16.1 to 0.17.0.

  • Server protocol version: 1.17.0 (from SDK 0.17.0)
  • Embedded agent client protocol: 1.13.0 (from the Codex Responses API stream)
  • Server check: "minor version difference must not exceed 1"
  • 1.17 - 1.13 = 4 → rejected with HTTP 500

Secondary: Retry storm → event loop starvation → Socket Mode death

With 41 unique sessions retrying every 5-15 seconds, this generated ~320 failed connections. The synchronous error handling consumed enough event loop time to miss Slack's 5-second pong deadline, causing Socket Mode disconnects and dropped replies.

Contributing: mem0 better-sqlite3 NODE_MODULE_VERSION mismatch

The com.smartclaw.gateway plist uses nvm's Node 22 (MODULE_VERSION 127). The better-sqlite3 module was compiled for Node 24 (MODULE_VERSION 137), adding error logging load.

Resolution

  1. Downgraded OpenClaw: 2026.3.28 → 2026.3.24
  2. Rebuilt better-sqlite3 using nvm's Node 22 node-gyp
  3. Updated meta.lastTouchedVersion to 2026.3.24
  4. Cleaned up plist conflict

Lessons Learned

  1. Check SDK versions before upgrading: npm view openclaw@<version> dependencies | grep agentclientprotocol
  2. checkCompatibility is NOT a valid top-level config key — it only exists in mem0 vectorStore config. Setting it at gateway/agents level crashes the gateway.
  3. The gateway plist Node binary matters: openclaw doctor creates a plist using whatever node is in PATH. Always check before rebuilding native modules.
  4. npm rebuild doesn't always recompile: Use node-gyp rebuild directly when prebuilt binaries don't match.
  5. ws-stream retries don't stop after session completion: A completed session's retry loop continues indefinitely — upstream bug.

Prevention

  • gateway-preflight.sh should verify SDK version compatibility before upgrades
  • Pre-start hook (tracked as orch-yps) should auto-rebuild native modules on NODE_MODULE_VERSION change
  • Monitor agent should alert on ws-stream 500 rates exceeding a threshold
